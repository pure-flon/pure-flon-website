MoE 앙상블로 크립토 트레이딩 시스템 만들기

Mixture of Experts (MoE) 아키텍처를 실전 크립토 트레이딩에 적용한 경험을 공유합니다. 4개의 전문 expert (Neutral-Long, S6-Long, Short, Flat)가 각각 다른 시장 레짐에서 활성화되며, gating network가 실시간으로 전문가를 선택합니다.

핵심 설계 결정:

  • 24차원 feature space (1시간 봉) — FR_z + OI_vel이 핵심 discriminator
  • Flat expert의 threshold를 0.35로 설정 — 관망이 가장 중요한 의사결정
  • Walk-forward 재학습: 고정 모델 PF 0.71(파산) vs Walk-forward PF 1.40(수익)
  • BOCD (Bayesian Online Change Detection)로 레짐 전환 감지 → 자동 재학습 트리거

결과: V4 라이브 배포 중. Long OOS PF 1.16, Short OOS PF 1.07. 자동 재학습 파이프라인으로 모델 드리프트 방어.

49개 AI 페르소나가 24시간 자율 운영하는 거버넌스 만들기

Claude Code로 49개의 전문 AI 페르소나를 만들고, 이들이 24시간 자율적으로 거버넌스를 수행하는 시스템을 구축했습니다. 각 페르소나는 고유한 권한, 경계, 하드가드를 가집니다.

핵심 아키텍처 (v2.0):

  • 병렬 DAG: 17 serial → 8 parallel (2.6x 가속, 신뢰도 16.7%→92.3%)
  • 에코 챔버 방지: 스펙트럴 분석으로 λ₁ < 0.50 강제 (외부 데이터 50%+ 필수)
  • Harness Engineer: graph/spectral/info 3-layer 메타 컨트롤러
  • Rate-Distortion 분석: 최적 페르소나 수 K=10±2

교훈: 시스템이 자기 자신을 유지보수하는 데 84.6% 시간을 쓰면 실패. 에코 브레이커가 핵심.

트레이딩 모델 자동 재학습 파이프라인 설계

고정 모델은 반드시 성능이 저하됩니다. 6년 백테스트에서 고정 모델 PF 0.71 (파산)이지만, Walk-forward 재학습 적용 시 PF 1.40 (수익). 자동 재학습 파이프라인의 핵심 설계를 공유합니다.

3종 트리거:

  • Weekly: 매주 월요일 07:00 UTC 정기 재학습
  • BOCD Drift: P(cp) > 0.05 + 24h Sharpe < 0 → 레짐 변화 감지
  • Performance Degradation: PF < 0.8 또는 5연속 손실

안전장치: 백업 3세대, survival gate (S+A≥20), 48h shadow 비교, 7일 쿨다운. 현재 모델 성능이 저하 중일 때만 자동 승격, 아니면 폐기.

Circuit Breaker FP 96% 문제 해결기

거버넌스 시스템의 Circuit Breaker가 False Positive 96%로 시스템을 마비시킨 사건. 93개 커밋이 push 불가 상태로 적체되고, 복구 커밋조차 push할 수 없는 데드락에 빠진 이야기.

근본 원인 3가지:

  • Gate validator가 미커밋 아티팩트에 실행 → 존재하지 않는 파일을 FAIL 처리
  • CB 에스컬레이션 쿨다운이 30분 → 17슬롯 순차 실행 중 3번 FP로 BLOCKED
  • BLOCKED 상태에서 복구 커밋 push 불가 → 데드락

해결: --committed-only 모드 (git ls-files), 쿨다운 4시간, recovery bypass (복구 파일만 BLOCKED에서도 push 허용), FP auto-recovery (BLOCKED→WARN 자동 강등).

자율 수익 탐색 엔진 (AREX) 설계

AI 시스템이 스스로 수익 기회를 탐색하고, 실험하고, 측정하고, Kill/Scale 판정을 내리는 자율 수익 탐색 엔진. 기존 스킬들의 "연결 코드"로 설계.

6-Phase Loop:

  • SCAN: market-radar로 기회 발굴 + 5-factor 점수
  • SELECT: 실험 설계 (가설, 성공 지표, Kill 기준)
  • BUILD: Supanova/Factory/gstack으로 MVP 생성
  • MEASURE: GitHub Traffic API로 무료 측정
  • DECIDE: Bayesian Kill/Scale 판정
  • LEARN: 카테고리별 Beta 사후분포 업데이트

Anti-echo 설계: 모든 KPI가 외부 (방문자, 클릭, 수익) → 자기 참조 불가능.
